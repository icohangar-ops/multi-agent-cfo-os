01 · The Problem

Open-Loop vs Closed-Loop Finance

Open Loop (Status Quo)
Senior controller carries 80% of institutional knowledge in their head
Variance commentary is written, read once, archived forever
Each layer of the org chart adds latency and loses signal
Audit trail is reconstructed from emails
Closed Loop (This System)
Decisions live in a structured Notion DB the LLM can query
Variance becomes a decision row that next month's run retrieves
Four agents replace translation; humans approve, don't transcribe
Every run writes an immutable, hash-stamped audit note

03 · Agent Pipeline

Four non-overlapping agents

Agent 1
Evidence
Loads every file under the period folder, parses, SHA-256 hashes, returns typed Evidence
Deterministic — No LLM
Agent 2
Analyst
Variance, cut-off, cash, and inventory analysis. Splits facts from assumptions
Gemini 2.5 Pro + Pandas
Agent 3
Memory
Retrieves prior decisions via Vector Search + Notion. Writes back ONLY after human approval
Gemini 2.5 Flash + Notion API
Agent 4
CFO Brief
Synthesizes close memo, 3 board messages, proposed Notion rows, and audit note
Gemini 2.5 Pro
04 · The Closed Loop

8 steps, every cycle

1
Read Evidence
2
Identify Signal
3
Separate Facts
4
Decide Action
5
Capture
6
Compare
7
Learn
8
Improve

This is not narrative. It is structural. The Memory Agent on cycle N+1 retrieves what the Memory Agent wrote on cycle N. That is the feedback signal the next brief is built on.

05 · Trust & Security

Built for auditability

Evidence Layer
SHA-256
Every file hashed; no LLM in the loading path
Analysis Layer
Pandas First
Numbers computed deterministically; LLM reasons over grounded summaries only
Human Gate
Approval
interrupt_before memory_write — no external write without explicit Y
Validation
JSON Schema
Typed Findings and CFOBrief objects; schema-checked LLM output
Audit Trail
Immutable
Hash-stamped audit note listing inputs, agents, outputs, decisions
Memory
Append-Only
Notion DB is append-only; corrections are new rows referencing prior IDs
